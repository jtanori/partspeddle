## Root Cause Analysis

### Primary Cause
Tests with lifecycle hooks (`beforeAll`, `afterAll`) that connect to Redis timeout after **10 seconds per hook** when Redis is unavailable. With multiple test files having such hooks, the cumulative delay creates the perception of a hang.

### Contributing Factors
1. **Docker not running:** `docker-compose.dev.yml` defines Postgres + Redis, but Docker daemon is offline
2. **Hook timeout too generous:** 10s timeout × many files = minutes of waiting
3. **No early-exit logic:** Tests don't skip when infrastructure is unavailable
4. **No infrastructure health check:** Test bootstrap doesn't verify services before running

### What This Is NOT
- ❌ Vitest runtime deadlock
- ❌ ESM/CJS boundary issue
- ❌ Module resolution recursion
- ❌ Vite configuration bug
- ❌ Next.js / Vitest incompatibility

## Resolution Strategy

### Option A: Infrastructure-First (Recommended)
1. Start Docker: `npm run infra:up`
2. Re-run tests
3. Verify deterministic execution

### Option B: Graceful Degradation
1. Add infrastructure health checks to test setup
2. Skip Redis/Postgres-dependent tests when services unavailable
3. Reduce hook timeout for faster failure

### Option C: Test Separation
1. Run pure unit tests separately: `vitest run src/backend/shared/queue`
2. Run integration tests only in CI with infrastructure
3. Frontend tests (T3.7) use MSW, no infrastructure needed
