# Governance Assimilation Waves

# (Recommended Kimi Rollout Sequence)

---

# WAVE 1 — Governance Root Normalization

---

Objective:
Create stable governance root structure.

Tasks:

* create /meta root
* create subsystem folders
* define naming conventions
* define schema versioning rules
* define canonical JSON principle
* define markdown projection principle

DO NOT:

* automate anything
* generate repository-wide metadata
* create topology systems
* enforce policies

Deliverables:

* directory structure
* base schemas
* governance README
* canonical-state definitions

Risk:
LOW

---

# WAVE 2 — Projection Architecture

---

Objective:
Separate canonical state from projections.

Tasks:

* define JSON source format
* define markdown projection format
* migrate existing project-knowledge conceptually
* create projection-generation conventions

DO NOT:

* fully automate projections yet
* rewrite all existing docs

Deliverables:

* projection architecture
* mapping conventions
* projection schemas

Risk:
LOW

---

# WAVE 3 — Runtime-Critical Registry

---

Objective:
Track only critical runtime systems.

Track:

* auth

* generate full dependency graphs

* subsystem registry
* ownership metadata
* criticality classification

Risk:
LOW-MEDIUM
