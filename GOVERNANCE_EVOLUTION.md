# 1. CORE PHILOSOPHY

---

VINTRACK is:

* an independent marketplace product

* a detachable construction substrate

* a transitional embedded governance architecture

SYNTH is:

* the future generalized governance platform
* extracted from validated governance primitives

The governance system MUST remain detachable from VINTRACK runtime architecture.

---

# 2. FOUNDATIONAL PRINCIPLES

---

## Construction Independence Principle (CIP)

The product runtime MUST remain independent from the governance system used to build it.

Governance MUST:

* remain detachable
* avoid runtime coupling
* avoid business logic ownership
* avoid deployment dependency

VINTRACK must remain operable if governance is removed entirely.

---

## JSON Canonical State Principle (JCSP)

JSON metadata is the canonical system state.

Markdown is:

* generated projection
* human-readable rendering
* operator-facing documentation

Markdown MUST NOT become authoritative state.

---

## Progressive Governance Principle (PGP)

Governance evolves progressively:

* informational
* advisory
* guided
* partially enforced
* selectively autonomous

Hard enforcement is prohibited during early integration phases.

---

## Runtime Critical First Principle (RCFP)

Governance onboarding begins ONLY with runtime-critical systems.

Initial scope:

* auth
* database
* deployment
* API contracts
* shared infrastructure
* runtime configuration

Non-critical systems onboard later.

---

## Governance Profile Principle (GPP)

Governance adapts to product type.

Governance profiles determine:

* schema sets
* workflow complexity
* enforcement level
* telemetry depth
* orchestration behavior

Governance MUST remain composable and context-sensitive.

---

## Structured Guided Governance Principle (SGGP)

Governance should default to templates and scaffolding.

Blank governance authoring is discouraged.

Templates MUST exist for:

* policies
* ADRs
* tickets
* milestones
* workflows
* blueprints
* execution reports
* recovery packets
